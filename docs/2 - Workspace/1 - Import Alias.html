Import Alias

To avoid import reference problems and even facilitate the implementation of complements and blueprints in the project, a series of aliases were created in the Typescript configuration, yet all packages were also configured so that it is not necessary to send unstable versions to NPM, making it possible to test how each package will work locally before publishing.

Below is the list of aliases previously configured in the project, to add a custom alias just access the tsconfig.json file.

@ucsjs/common Import all scripts from the `packages/common` directory
@ucsjs/core Import all scripts from the `packages/core` directory
@ucsjs/blueprints Import all scripts from the `packages/blueprints` directory
@ucsjs/repository Import all scripts from the `packages/repository` directory
@abstracts Import all scripts from the `src/abstracts` directory
@blueprints Import all scripts from the `src/blueprints` directory
@controllers Import all scripts from the `src/controllers` directory
@core Import all scripts from the `src/core` directory
@decorators Import all scripts from the `src/decorators` directory
@dtos Import all scripts from the `src/dtos` directory
@enums Import all scripts from the `src/enums` directory
@interfaces Import all scripts from the `src/interfaces` directory
@modules Import all scripts from the `src/modules` directory
@services Import all scripts from the `src/services` directory
